# Sprint Report Template

Use when posting a cycle completion summary to a Plane cycle (`description_html` field on a cycle),
or as a standalone Plane page for sprint retrospectives.

## Template

```html
<h2>Sprint {cycle_name} — {start_date} to {end_date}</h2>

<h3>Summary</h3>
<p>{2-3 sentences: what the team shipped, overall health, and tone of the sprint}</p>

<h3>Velocity</h3>
<table>
  <thead>
    <tr>
      <th>Metric</th>
      <th>Value</th>
    </tr>
  </thead>
  <tbody>
    <tr><td>Issues committed</td><td>{committed_count}</td></tr>
    <tr><td>Issues completed</td><td>{completed_count}</td></tr>
    <tr><td>Completion rate</td><td>{pct}%</td></tr>
    <tr><td>Story points committed</td><td>{sp_committed}</td></tr>
    <tr><td>Story points completed</td><td>{sp_completed}</td></tr>
  </tbody>
</table>

<h3>✅ Highlights — What Shipped</h3>
<ul>
  <li><a href="{issue_url}">{issue_title}</a> — {1-line impact}</li>
  <li><a href="{issue_url}">{issue_title}</a> — {1-line impact}</li>
</ul>

<h3>🚫 Blockers &amp; Carry-overs</h3>
<ul>
  <li><a href="{issue_url}">{issue_title}</a> — {reason not completed}</li>
</ul>

<h3>📊 Burn-down Snapshot</h3>
<p>{Narrative of how issues progressed over the sprint. E.g.: "Front-loaded with 60% closed by day 3;
stalled mid-sprint due to infra blocker; recovered last 2 days."}</p>

<h3>🔍 Retrospective Notes</h3>
<ul>
  <li><strong>What went well:</strong> {observation}</li>
  <li><strong>What to improve:</strong> {observation}</li>
  <li><strong>Action item:</strong> {concrete next step with owner}</li>
</ul>

<h3>Next Sprint Preview</h3>
<ul>
  <li>{planned focus area or top priority issue for next cycle}</li>
</ul>

<hr>
<p><sub>Report generated by {agent_name} · Cycle: <a href="{cycle_url}">{cycle_name}</a></sub></p>
```

## Placeholder Guide

| Placeholder | What to fill |
|---|---|
| `{cycle_name}` | Plane cycle name, e.g. `Sprint 4` |
| `{start_date}` / `{end_date}` | ISO dates: `2026-03-17` / `2026-03-30` |
| `{committed_count}` | Total issues in cycle at start |
| `{completed_count}` | Issues moved to Done state |
| `{pct}` | `completed / committed * 100` rounded |
| `{sp_committed}` / `{sp_completed}` | Story points (omit row if not tracked) |
| `{issue_url}` | Plane issue URL — resolve with fleet-urls skill |
| `{cycle_url}` | Plane cycle URL |

## Minimal Variant (no story points tracked)

Remove the story-point rows from the Velocity table and the burn-down section if point data is unavailable.
Replace with a plain issue count summary.

## Notes on HTML Entities

- Use `&amp;` for `&` in text (e.g., `Blockers &amp; Carry-overs`)
- Use `&gt;` and `&lt;` inside text nodes if needed
- Plane's editor generally handles these correctly
