# Layer 6: fleet-context.md

**Type:** Context File (Outer layer — dynamic, refreshed every 30s)
**Position:** 6 of 8 (fleet state — agent sees current situation)
**System:** S07 (Orchestrator), S19 (Session), S22 (Agent Intelligence)
**Standard:** context-files-standard.md
**Location:** agents/{name}/context/fleet-context.md

## Purpose

Current fleet state and role-specific data. GENERATED by the orchestrator every 30 seconds. Agent wakes up and sees: what mode the fleet is in, what the PO directs, who's talking to them, what tasks they have, what happened since last heartbeat. FULL data, NEVER compressed (PO requirement — the 300-char compression disease made agents non-functional).

## Required Sections (7, in order)

1. **Fleet State** — work_mode, cycle_phase, backend_mode, budget_mode, agents online/total
2. **PO Directives** — active directives for this agent or "all" (HIGHEST PRIORITY)
3. **Messages** — @mentions since last heartbeat (full content, not truncated)
4. **Assigned Tasks** — FULL task details: title, verbatim, stage, readiness, progress, artifacts, comments
5. **Role Data** — role-specific:
   - PM: unassigned tasks, sprint metrics, pending gates
   - Fleet-ops: pending approvals, review queue, offline agents
   - Architect: tasks needing design review, high complexity tasks
   - DevSecOps: PRs needing security review, security alerts
   - Workers: artifact state, completeness %, contributions received
6. **Events** — relevant events since last heartbeat (filtered by tag subscriptions)
7. **Context Awareness** — context usage %, rate limit usage % (both countdowns)

## Generation Pipeline

```
Orchestrator Step 0: _refresh_agent_contexts()
├── mc.list_tasks(board_id)           → all tasks (cached per cycle)
├── mc.list_memory(board_id)          → board memory (directives, mentions)
├── For each agent:
│   ├── Filter tasks assigned to this agent
│   ├── Filter messages mentioning this agent
│   ├── Filter PO directives for this agent
│   ├── role_providers.py:
│   │   ├── fleet_ops_provider() → approvals, review queue, offline
│   │   ├── pm_provider() → unassigned, blocked, sprint progress
│   │   ├── architect_provider() → design-needed tasks
│   │   ├── devsecops_provider() → security tasks, PRs for review
│   │   └── worker_provider() → task count, artifact state
│   ├── preembed.py: format_heartbeat_preembed()
│   └── context_writer.py: write_heartbeat_context()
└── File written: agents/{name}/context/fleet-context.md
```

## Relationships

- GENERATED BY: orchestrator.py Step 0, preembed.py, context_writer.py, role_providers.py
- INJECTED BY: gateway (_build_agent_context, position 6, max 8000 chars per context file)
- READ BY: agent at every heartbeat/dispatch
- REFRESHED: every orchestrator cycle (scales with budget_mode: 5s turbo → 60s economic)
- DATA FROM: mc_client.py (tasks, memory, approvals), plane_client.py (sprint data for PM)
- FEEDS: agent's situational awareness (knows what's happening without calling tools)
- CONNECTS TO: knowledge map navigator (map-driven content can be merged with fleet state)
- CONNECTS TO: SessionStart hook (hook can augment with additional context)
- DOES NOT CONTAIN: identity (Layer 1), values (Layer 2), rules (Layer 3), tool docs (Layer 4), colleague info (Layer 5), action protocol (Layer 8)
- DYNAMIC: changes every 30s — the only layer that changes during operation (along with Layer 7)
- PER AGENT: 10 unique fleet-context.md files (role-specific data)
