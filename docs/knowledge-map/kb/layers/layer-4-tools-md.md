# Layer 4: TOOLS.md

**Type:** Agent File (Middle layer — stable, GENERATED)
**Position:** 4 of 8 (capabilities — agent knows tool consequences before deciding)
**System:** S08 (MCP Tools), S22 (Agent Intelligence)
**Standard:** tools-agents-standard.md

## Purpose

Chain-aware tool reference. NOT a list of tool names — documents what HAPPENS when each tool is called. Each tool entry has 5 required fields: What (primary action), When (stage/conditions), Chain (automatic downstream effects), Input (key parameters), You do NOT need to (what the chain handles automatically). Agent knows consequences BEFORE calling a tool.

## Generation

GENERATED by `scripts/generate-tools-md.sh` — never hand-written.
Sources: fleet/mcp/tools.py (tool definitions) + fleet-elevation/24 (call trees) + config/agent-tooling.yaml (per-role assignments)

Different per role — architect gets different tools than engineer. Only tools the role ACTUALLY uses are documented. No extra tools.

## Required Format Per Tool (5 fields)

```
### fleet_{tool_name}
**What:** primary action (one line)
**When:** stage/conditions for use
**Chain:** automatic downstream effects (events, notifications, Plane sync, trail)
**Input:** key parameters
**You do NOT need to:** what the chain handles (prevents manual duplication)
```

## Relationships

- INJECTED BY: gateway (_build_agent_context, position 4)
- GENERATED BY: scripts/generate-tools-md.sh
- SOURCE: fleet/mcp/tools.py (tool implementations), fleet-elevation/24 (call tree catalog)
- CONFIGURED BY: config/agent-tooling.yaml (which tools per role)
- VALIDATED BY: scripts/validate-agents.sh (all 5 fields per tool, tool names match code, chain descriptions match call trees, stage restrictions documented)
- FEEDS: agent's tool usage decisions (knows what happens before calling)
- CONNECTS TO: CLAUDE.md Layer 3 (tool chains section references same tools more briefly)
- CONNECTS TO: MCP server (fleet MCP provides the actual tools)
- CONNECTS TO: stage_context.py (stage restrictions per tool)
- DOES NOT CONTAIN: identity, values, rules, colleague info, dynamic data, action protocol
- STABLE: regenerated when tools change, not during operation
- PER AGENT: 10 unique TOOLS.md files (different tools per role)
