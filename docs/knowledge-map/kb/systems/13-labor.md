## What This System Does

Nothing produced by the fleet is anonymous. Every task completion gets a LaborStamp recording: which agent, which backend (claude-code/localai/openrouter/direct), which model (opus/sonnet/hermes-3b), what confidence tier (expert/standard/trainee/community — auto-derived from backend+model), cost, duration, tokens, lines changed, skills used, challenge rounds survived.

Confidence tier auto-derivation:
- claude-code + opus → expert
- claude-code + sonnet → standard
- localai + any → trainee
- openrouter-free + any → community

**Trainee and community tier automatically set `requires_challenge = True`** — fleet-ops cannot approve without adversarial challenge passing first.

## Stamp Assembly

At fleet_task_complete time:
```
DispatchRecord (saved at dispatch) + session_telemetry data
  → assemble_stamp() → LaborStamp
  → Write 8+ labor_* fields to task custom fields
  → Include in completion comment (table format)
  → Include in PR body (References section)
  → Emit fleet.labor.recorded event
```

DispatchRecord saved at dispatch: task_id, agent_name, backend, model, effort, selection_reason, skills, dispatched_at. Used to calculate duration (now - dispatched_at).

## Heartbeat Cost Tracking

heartbeat_stamp.py separately tracks idle fleet cost:
- Per-agent heartbeat average cost
- Anomaly detection (>2× average → alert)
- Total heartbeat cost as % of fleet cost
- Alert if heartbeats > 30% of total (the March catastrophe was mostly heartbeat drain)

HeartbeatCostTracker: rolling per-agent metrics, anomaly detection thresholds.

## Labor Analytics

labor_analytics.py: rolling window of 500 stamps, per-agent/model/tier metrics, cost breakdowns by backend and budget mode, approval rates per tier.
