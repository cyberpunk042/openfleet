# Playwright MCP Server

**Type:** External MCP Server
**Package:** @playwright/mcp@latest
**Transport:** npx (stdio)
**Tools:** navigate, click, type, snapshot, screenshot, evaluate, wait
**Installed for:** software-engineer, qa-engineer, ux-designer

## What It Does

Headless browser automation via Playwright. Navigate to URLs, click elements, type text, take snapshots/screenshots, evaluate JavaScript, wait for selectors. Runs in headless mode for automated testing and UI interaction.

## Fleet Use Case

QA runs E2E tests and validates UI behavior. Engineer tests frontend changes. UX designer validates accessibility and visual design. All use Playwright for browser-based validation.

## Relationships

- INSTALLED FOR: software-engineer, qa-engineer, ux-designer
- CONNECTS TO: quality-accessibility skill (UX uses Playwright for accessibility testing)
- CONNECTS TO: feature-test skill (E2E tests via Playwright)
- CONNECTS TO: fleet-test skill (test result analysis after Playwright runs)
