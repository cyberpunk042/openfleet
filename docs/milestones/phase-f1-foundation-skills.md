# Phase F1: Foundation Skills — Making Agents Smart and Work Easy

> "all I said about making it smart and tooling for the agent and making it easy... you have to make things fluid and clear and tag things and make sure that the flow everywhere at all level is top notch"

## Problem Statement

Agents currently produce bare-minimum output. PRs are text dumps with a basic template.
Task comments are unstructured. Board memory is underused. IRC messages are plain text.
Cross-references are manual. Nothing is visually appealing. Nothing exploits markdown.

The human produces better PRs with Copilot than the fleet produces autonomously.
That's unacceptable.

## Design Principle

**Skills handle the complexity, agents focus on the work.**

An agent should call a skill and get publication-quality output. The agent doesn't need
to think about formatting, linking, or structure — the skill handles it. This is what
"making it easy" means.

## Architecture

### URL Builder (M81)

**Purpose:** Given a project, branch, file, or task — produce the correct clickable URL.

**Why a skill and not just a config?** Because agents need to call it inline while
composing messages. A skill can be invoked contextually: "I'm working on NNRT,
branch fleet/sw-eng/3402f526, file nnrt/core/engine.py — what are my URLs?"

**Design:**

```
Input:
  project: nnrt
  branch: fleet/software-engineer/3402f526
  files: [nnrt/core/engine.py, nnrt/core/contracts.py]
  task_id: 3402f526-78c2-455f-b7fb-f21765d67593
  pr_number: 3

Output:
  task_url: http://localhost:3000/boards/828d80ab/tasks/3402f526
  pr_url: https://github.com/cyberpunk042/Narrative-to-Neutral-Report-Transformer/pull/3
  compare_url: https://github.com/cyberpunk042/Narrative-to-Neutral-Report-Transformer/compare/main...fleet/software-engineer/3402f526
  file_urls:
    - https://github.com/cyberpunk042/Narrative-to-Neutral-Report-Transformer/blob/fleet/software-engineer/3402f526/nnrt/core/engine.py
    - https://github.com/cyberpunk042/Narrative-to-Neutral-Report-Transformer/blob/fleet/software-engineer/3402f526/nnrt/core/contracts.py
```

**Implementation approach:**
- Config file: `config/url-templates.yaml` with project→repo mapping and URL patterns
- Skill: `fleet-urls` SKILL.md that reads the config and resolves URLs
- Script: `scripts/resolve-urls.sh` that other scripts can call
- The skill teaches agents HOW to use it and WHEN to include URLs

**Dependencies:** config/projects.yaml (already has project names, needs GitHub owner/repo added)

**Purpose:** Reusable templates for every type of output agents produce.

**Templates needed:**

### PR Composer Skill (M83)

**Purpose:** Takes a branch diff and produces a publication-quality PR.

> "the PR I usually generate myself via copilot or claude code are much more elevated"

**What the skill does:**
1. Agent calls the skill when ready to create a PR
2. Skill reads the git log and diff from the worktree
3. Generates changelog from conventional commits (grouped by type)
4. Creates file-by-file change descriptions
5. Resolves all URLs (task, compare, files)
6. Fills the PR body template
7. Creates the PR via `gh pr create`

**What makes it "elevated":**
- Changelog is auto-generated, not manually written
- Every file has a description, not just a path
- Every reference is a clickable link
- Visual structure with emoji, tables, checklists
- Footer with agent attribution and task reference
- Verification checklist for reviewer

**This skill REPLACES the raw `gh pr create` call in the agent workflow.** Instead of agents writing their own PR body, they invoke the skill which handles everything.

### Task Comment Formatter (M84)

**Purpose:** Agents call this to produce properly formatted task comments.

**Design:** The skill provides templates for each comment type (acceptance, progress, completion, blocker). Agent fills in the content, skill handles the markdown, URLs, and structure.

**What changes:** Instead of agents manually composing curl commands with raw JSON, they invoke the skill which:
1. Reads TOOLS.md for credentials
2. Takes structured input (status, summary, files, etc.)
3. Selects the right template
4. Resolves all URLs
5. Posts via MC API

This makes it EASY for agents — they don't think about formatting.

---

### Board Memory Composer (M85)

**Purpose:** Proper tagged, formatted board memory entries.

> "tag things and make sure that the flow everywhere at all level is top notch"

**Design:** The skill provides templates for each memory type (alert, decision, suggestion, knowledge). Ensures proper tags, proper formatting, proper cross-references.

**Tags taxonomy:**
- Type: alert, decision, suggestion, knowledge, report
- Severity: critical, high, medium, low
- Project: project:nnrt, project:fleet, project:aicp
- Area: security, architecture, quality, workflow, tooling
- Agent: agent:architect, agent:software-engineer, etc.

---

### IRC Message Formatter (M86)

**Purpose:** Structured IRC messages with URLs.

> "warn me about important CVE or security issue in general or bug or suggestion or fix or blocking or issue"

**Design:** The skill formats messages for different event types:
- Task events: created, dispatched, completed, blocked
- PR events: ready, merged, failed
- Alerts: security, quality, architecture
- Agent status: online, offline, error

Each format includes the right URLs and emoji for scannability.

## Dependencies Between Skills

```
config/url-templates.yaml (project → repo mapping)
    ↓
fleet-urls skill (M81) — resolves URLs
    ↓
markdown templates (M82) — reference URLs
    ↓
fleet-pr skill (M83) — uses templates + URLs
fleet-comment skill (M84) — uses templates + URLs
fleet-memory skill (M85) — uses templates + URLs
fleet-irc skill (M86) — uses URLs
```

M81 and M82 are the foundation. Everything else builds on them.

## Implementation Order

1. **M81**: URL templates config + fleet-urls skill
2. **M82**: Markdown template library in `agents/_template/markdown/`
3. **M83**: fleet-pr skill (replaces raw `gh pr create`)
4. **M84**: fleet-comment skill (replaces raw curl to MC API)
5. **M85**: fleet-memory skill (replaces raw curl to board memory)
6. **M86**: fleet-irc skill (enhances notify-irc.sh)
